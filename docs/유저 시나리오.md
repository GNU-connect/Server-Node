# 사용자 시나리오

## 프로젝트 개요

커넥트 지누는 경상국립대 사용자들이 카카오톡 챗봇과 모바일 앱에서 학교 생활에 필요한 정보를 빠르게 확인할 수 있도록 돕는 서비스이다.

- 카카오톡 챗봇에서는 공지사항, 학식, 학사일정, 셔틀버스, 내 정보 조회를 이용할 수 있다.
- 모바일 앱에서는 캠퍼스별 학식과 셔틀버스 정보를 확인할 수 있다.
- 시스템은 학교에서 제공되는 정보를 모아 사용자가 보기 쉬운 형태로 보여준다.

---

## 등장인물 (Actors)

| 역할              | 설명                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| **사용자**        | 커넥트 지누를 통해 교내 정보를 확인하는 사람                         |
| **카카오톡 챗봇** | 사용자가 카카오톡 안에서 교내 정보를 조회할 수 있는 대화형 서비스    |
| **모바일 앱**     | 사용자가 앱 화면에서 학식과 셔틀 정보를 조회할 수 있는 서비스        |
| **시스템**        | 사용자의 요청에 맞는 교내 정보를 찾아 응답하는 서비스                |
| **교내 정보**     | 공지사항, 식단, 학사일정, 셔틀 시간표, 열람실 좌석 등 학교 생활 정보 |

---

## 시나리오 1: 사용자 설정

### 1-1. 사용자가 카카오톡 챗봇을 처음 사용한다

**흐름:**

1. 사용자가 챗봇에서 원하는 기능을 선택한다.
2. 시스템은 사용자가 누구인지 확인한다.
3. 이전에 저장한 캠퍼스와 학과 정보가 있으면 해당 정보를 기준으로 서비스를 제공한다.
4. 저장한 정보가 없으면 캠퍼스와 학과가 아직 등록되지 않은 사용자로 처리한다.

**예외:**

- 처리 중 문제가 발생한 경우 → “예상치 못한 오류가 발생했어!” 메시지를 보여준다.

---

### 1-2. 사용자가 내 프로필을 조회한다

**흐름:**

1. 사용자가 내 프로필 조회를 선택한다.
2. 시스템은 저장된 캠퍼스와 학과 정보를 확인한다.
3. 챗봇은 현재 등록된 내 정보를 보여준다.

**예외:**

- 등록된 정보가 없는 경우 → 캠퍼스와 학과가 미설정 상태로 표시된다.

---

### 1-3. 사용자가 캠퍼스와 학과를 등록하거나 변경한다

**흐름:**

1. 사용자가 캠퍼스/학과 설정을 선택한다.
2. 챗봇은 캠퍼스 목록을 보여준다.
3. 사용자가 캠퍼스를 선택한다.
4. 챗봇은 단과대학 목록을 한 번에 최대 5개씩 보여준다.
5. 사용자가 단과대학을 선택한다.
6. 챗봇은 학과 목록을 한 번에 최대 5개씩 보여준다.
7. 사용자가 학과를 선택한다.
8. 시스템은 선택한 캠퍼스와 학과를 내 정보로 저장한다.
9. 챗봇은 “학과 정보를 등록했어!” 메시지를 보여준다.

**예외:**

- 선택한 캠퍼스나 학과 정보를 찾을 수 없는 경우 → 오류 메시지를 보여준다.

**표시 기준:**

- 단과대학과 학과 목록은 한 화면에 최대 5개씩 보여준다.
- 항목이 5개를 넘는 경우 이전/다음 선택으로 다른 항목을 확인할 수 있다.

---

## 시나리오 2: 공지사항 조회

### 2-1. 사용자가 학교 공지사항을 조회한다

**흐름:**

1. 사용자가 학교 공지사항 조회를 선택한다.
2. 시스템은 학교 전체 공지 중 주요 카테고리의 최신 글을 찾는다.
3. 챗봇은 카테고리별 공지사항을 카드 형태로 보여준다.
4. 사용자는 관심 있는 공지사항을 눌러 원문을 확인할 수 있다.

**예외:**

- 보여줄 공지사항이 없는 경우 → “현재 등록된 공지사항이 없어!” 메시지를 보여준다.

**표시 기준:**

- 학교 공지사항은 `기관`, `채용`, `장학`, `외부기관 행사`, `학사` 5개 카테고리를 보여준다.
- 각 카테고리마다 최신 공지사항을 최대 4개씩 보여준다.
- 전체로는 최대 20개의 공지사항을 보여준다.

---

### 2-2. 사용자가 내 학과 공지사항을 조회한다

**사전 조건:**

- 사용자의 학과 정보가 등록되어 있다.

**흐름:**

1. 사용자가 학과 공지사항 조회를 선택한다.
2. 시스템은 내 학과의 최신 공지사항을 찾는다.
3. 필요한 경우 관련 상위 학과의 공지사항도 함께 찾는다.
4. 챗봇은 학과 공지사항을 카드 형태로 보여준다.
5. 사용자는 관심 있는 공지사항을 눌러 원문을 확인할 수 있다.

**예외:**

- 학과 정보가 등록되어 있지 않은 경우 → 학과를 먼저 등록하도록 안내한다.
- 보여줄 공지사항이 없는 경우 → “현재 등록된 공지사항이 없어!” 메시지를 보여준다.

**표시 기준:**

- 학과 공지사항은 공지 분류별로 최신 글을 최대 4개씩 보여준다.
- 관련 상위 학과 공지사항이 함께 있는 경우에도 각 분류별 최대 4개 기준을 유지한다.

---

## 시나리오 3: 학식 조회

### 3-1. 사용자가 챗봇에서 식당 목록을 조회한다

**흐름:**

1. 사용자가 학식 조회를 선택한다.
2. 시스템은 사용자가 선택한 캠퍼스가 있는지 확인한다.
3. 선택한 캠퍼스가 없으면 저장된 내 캠퍼스 정보를 확인한다.
4. 챗봇은 해당 캠퍼스의 식당 목록을 보여준다.
5. 사용자는 원하는 식당을 선택한다.

**예외:**

- 선택한 캠퍼스도 저장된 캠퍼스도 없는 경우 → 캠퍼스를 먼저 선택하도록 안내한다.
- 해당 캠퍼스에 등록된 식당이 없는 경우 → 식당 정보가 없다고 안내한다.

**표시 기준:**

- 식당 목록은 선택한 캠퍼스에 등록된 식당을 보여준다.
- 다른 캠퍼스의 식당을 보고 싶으면 캠퍼스 선택으로 이동할 수 있다.

---

### 3-2. 사용자가 챗봇에서 특정 식당의 식단을 조회한다

**사전 조건:** 사용자가 식당을 선택했다.

**흐름:**

1. 사용자가 날짜와 끼니를 선택한다.
2. 시스템은 선택한 식당의 해당 날짜, 해당 끼니 메뉴를 찾는다.
3. 끼니를 선택하지 않은 경우 현재 시간에 맞는 끼니를 기준으로 메뉴를 찾는다.
4. 챗봇은 식당명, 날짜, 끼니, 메뉴를 보여준다.

**예외:**

- 선택한 식당 정보를 찾을 수 없는 경우 → 오류 메시지를 보여준다.
- 해당 날짜와 끼니에 등록된 메뉴가 없는 경우 → 메뉴가 없다고 안내한다.

**표시 기준:**

- 챗봇에서는 `오늘 아침`, `오늘 점심`, `오늘 저녁`, `내일 아침`, `내일 점심`, `내일 저녁` 총 6개의 빠른 선택지를 제공한다.
- 선택한 식당의 메뉴는 분류별로 묶어서 보여준다.

**자동 선택 기준:**

- 끼니를 직접 선택하지 않은 경우 한국 시간 기준으로 자동 선택한다.
- 19:00부터 다음 날 09:29까지는 `아침`으로 본다.
- 09:30부터 13:29까지는 `점심`으로 본다.
- 13:30부터 18:59까지는 `저녁`으로 본다.
- 날짜도 직접 선택하지 않은 경우 19:00 전에는 `오늘`, 19:00 이후에는 `내일` 메뉴를 보여준다.

---

### 3-3. 사용자가 모바일 앱에서 학식을 조회한다

**흐름:**

1. 앱은 캠퍼스 목록을 불러온다.
2. 앱은 기본 캠퍼스를 선택해 식당 목록을 보여준다.
3. 앱은 현재 시간에 맞는 끼니를 먼저 선택해 둔다.
4. 사용자가 캠퍼스를 바꾸면 해당 캠퍼스의 식당 목록으로 바뀐다.
5. 사용자가 식당, 날짜, 끼니를 선택한다.
6. 앱은 선택한 조건에 맞는 메뉴를 보여준다.

**예외:**

- 캠퍼스 정보를 불러오지 못한 경우 → “캠퍼스 정보를 불러오지 못했습니다.” 메시지를 보여준다.
- 식당 정보를 불러오지 못한 경우 → “식당 정보를 불러오지 못했습니다.” 메시지를 보여준다.
- 메뉴 정보를 불러오지 못한 경우 → “메뉴 정보를 불러오지 못했습니다.” 메시지를 보여준다.
- 등록된 메뉴가 없는 경우 → “등록된 메뉴가 없어요” 메시지를 보여준다.

**표시 기준:**

- 앱에서는 오늘부터 7일간의 날짜를 선택할 수 있다.
- 끼니는 아침, 점심, 저녁 중 하나를 선택한다.
- 메뉴는 분류별로 묶어서 보여준다.

**자동 선택 기준:**

- 앱을 처음 열면 휴대폰에 설정된 현재 시간 기준으로 끼니를 자동 선택한다.
- 19:00부터 다음 날 09:29까지는 `아침`으로 본다.
- 09:30부터 13:29까지는 `점심`으로 본다.
- 13:30부터 18:59까지는 `저녁`으로 본다.

---

## 시나리오 4: 학사일정 조회

### 4-1. 사용자가 챗봇에서 특정 월의 학사일정을 조회한다

**흐름:**

1. 사용자가 학사일정 조회를 선택한다.
2. 사용자가 조회할 월을 선택한다.
3. 시스템은 선택한 월의 학사일정을 찾는다.
4. 챗봇은 해당 월의 학사일정을 보여준다.

**예외:**

- 올바르지 않은 월을 선택한 경우 → “올바른 월을 입력해주세요. (1-12)” 메시지를 보여준다.
- 해당 월에 등록된 학사일정이 없는 경우 → 일정이 없다고 안내한다.

---

### 4-2. 사용자가 챗봇에서 월을 선택하지 않고 학사일정을 조회한다

**흐름:**

1. 사용자가 학사일정 조회를 선택한다.
2. 월을 따로 선택하지 않은 경우 시스템은 이번 달 학사일정을 찾는다.
3. 챗봇은 이번 달 학사일정을 보여준다.

---

## 시나리오 5: 셔틀버스 조회

### 5-1. 사용자가 챗봇에서 셔틀 노선 목록을 조회한다

**흐름:**

1. 사용자가 셔틀버스 조회를 선택한다.
2. 시스템은 등록된 셔틀 노선을 찾는다.
3. 챗봇은 노선 목록을 보여준다.
4. 사용자는 원하는 노선을 선택한다.

**예외:**

- 등록된 노선이 없는 경우 → 노선 정보가 없다고 안내한다.

**표시 기준:**

- 셔틀 노선 목록은 현재 제공 중인 2개 노선을 보여준다.
- 제공 노선은 `가좌 → 칠암 캠퍼스`, `칠암 → 가좌 캠퍼스`이다.

---

### 5-2. 사용자가 챗봇에서 셔틀 시간표를 조회한다

**사전 조건:** 사용자가 셔틀 노선을 선택했다.

**흐름:**

1. 사용자가 특정 노선을 선택한다.
2. 시스템은 해당 노선의 시간표를 찾는다.
3. 챗봇은 선택한 노선의 시간표를 보여준다.

**예외:**

- 선택한 노선 정보를 찾을 수 없는 경우 → 오류 메시지를 보여준다.

**표시 기준:**

- 선택한 노선의 시간표는 구간별로 묶어 보여준다.
- 금요일에 운행하지 않는 시간은 별도 표시로 구분한다.

---

### 5-3. 사용자가 모바일 앱에서 셔틀 시간표와 다음 버스를 조회한다

**흐름:**

1. 사용자가 하단 메뉴에서 셔틀 화면을 선택한다.
2. 앱은 셔틀 노선 목록을 보여준다.
3. 앱은 첫 번째 노선을 기본으로 선택한다.
4. 사용자가 다른 노선을 선택하면 해당 노선의 시간표로 바뀐다.
5. 앱은 현재 시간 기준으로 가장 가까운 다음 버스를 보여준다.
6. 앱은 이미 지난 시간, 다음 버스 시간, 앞으로 남은 시간을 구분해 보여준다.
7. 앱은 일정 시간마다 시간표를 새로 불러와 최신 상태를 유지한다.
8. 사용자는 갱신 버튼을 눌러 시간표를 바로 새로 불러올 수 있다.

**예외:**

- 노선 정보를 불러오지 못한 경우 → “노선 정보를 불러오지 못했습니다.” 메시지를 보여준다.
- 시간표를 불러오지 못한 경우 → “시간표를 불러오지 못했습니다.” 메시지를 보여준다.
- 오늘 남은 운행이 없는 경우 → “오늘 운행이 종료되었습니다” 메시지를 보여준다.

**표시 기준:**

- 다음 버스는 현재 시간 이후 가장 가까운 1개만 강조해서 보여준다.
- 전체 시간표는 구간별로 묶어 보여준다.
- 시간표는 1분마다 자동으로 새로 확인한다.
