## 코드 품질

- TypeScript(JavaScript) 파일을 수정한 후에는 반드시 'pnpm test' 명령어를 실행하여 테스트를 실행합니다.
- 테스트 통과 후에는 'pnpm lint' 명령어를 실행하여 코드 스타일을 검사합니다.

## 커밋

- 커밋 메시지 포맷은 `[gitmoji] [type]: [subject]` 형식으로 작성합니다.
  - 예: `🔧 chore: 모바일 앱 환경 설정 변경`
- 커밋 메시지는 항상 한글로 작성합니다.
- subject는 간결하게 변경 내용을 요약합니다. (한 줄)
- 불필요한 조사나 장황한 표현은 지양합니다.
- 커밋 메시지 바디는 필요할 경우에만 작성합니다.
- 아래 내용을 포함할 수 있습니다:
  - 변경 이유
  - 주요 변경 사항 상세
  - 참고 사항 (이슈 번호 등)
- 예시:

```
✨ feat: 사용자 로그인 기능 추가

- JWT 기반 인증 로직 구현
- 로그인 실패 시 에러 메시지 처리 개선
- 관련 이슈: #12
```
